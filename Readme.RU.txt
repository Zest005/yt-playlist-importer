┌─────────────────────────────────────────────────────────────────┐
│  ДЛЯ ЧЕГО ЭТОТ СКРИПТ                                           │
└─────────────────────────────────────────────────────────────────┘

Переносит плейлисты со старого YouTube-аккаунта в новый.

YouTube/Google не предоставляет встроенного инструмента импорта
плейлистов. Если вы хотите перенести свои плейлисты на новый
аккаунт, единственный официальный способ — выгрузить их через
Google Takeout в виде CSV-файлов, а дальше создавать всё руками.

Этот скрипт автоматизирует «дальше». Он берёт CSV-файлы из
Takeout, создаёт плейлисты в текущем (целевом) аккаунте YouTube
и наполняет их видео — через тот же внутренний API, которым
пользуется сам сайт youtube.com.

Не требует:
  • API-ключей или регистрации приложения в Google Cloud
  • OAuth-токенов
  • установки чего-либо
  • загрузки данных на сторонние сервисы

Всё работает локально в браузере, на cookie вашей текущей
сессии YouTube.

Интерфейс на трёх языках: EN / RU / UA. Выбор языка — кнопками
в правом верхнем углу окна. Язык автоматически определяется по
языку браузера и запоминается между запусками.


┌─────────────────────────────────────────────────────────────────┐
│  ВОЗМОЖНОСТИ                                                    │
└─────────────────────────────────────────────────────────────────┘

  • Загрузка одного или нескольких CSV-файлов через drag-and-drop
    или диалог выбора файлов.
  • Создание плейлистов с произвольными именами (берутся из имени
    файла Takeout).
  • Автораспознавание «Watch later» / «Смотреть позже» /
    «Дивитися пізніше» — такие файлы попадают в системный
    плейлист «Смотреть позже», а не создают одноимённый обычный.
  • Опциональная очистка «Смотреть позже» перед импортом —
    галочка в настройках. Скрипт постранично удалит ВСЕ
    текущие видео из WL, потом наполнит его из CSV.
  • Для Watch Later видео из CSV добавляются в обратном порядке,
    чтобы отображение в новом WL совпадало с оригиналом
    (подробнее ниже в разделе «Watch Later — особенности»).
  • Настройка приватности создаваемых плейлистов:
        — Приватный (только вы)
        — По ссылке (Unlisted)
        — Публичный
  • Настройка задержки между запросами (от 100 мс до 60 секунд),
    с готовыми пресетами: 0.5 / 0.7 / 3 / 10 / 15 секунд.
  • Настройка размера батча (сколько видео отправлять одним
    запросом, 1–50).
  • Прогресс-бар с оценкой оставшегося времени (ETA), счётчиком
    добавленных видео и живым логом операций.
  • Кнопка остановки импорта в любой момент.
  • Автоматический повтор по одному видео, если батч не прошёл
    целиком — чтобы одно битое видео не «уронило» 19 соседей.
  • Обработка rate-limit (HTTP 429): автопауза 30–60 секунд
    с продолжением.
  • Различение типов ошибок:
        ⊘ «недоступно» — видео удалено, приватно, заблокировано
          в регионе, имеет возрастное ограничение, или автор
          отключил добавление в плейлисты (HTTP 400, FAILED_PRECONDITION)
        ✗ «ошибка»     — всё остальное (сеть, неизвестные коды)
  • Подробный финальный отчёт прямо в окне:
        — общая статистика (добавлено / недоступны / ошибки)
        — разбивка по плейлистам
        — список не добавленных видео с кликабельными ссылками
        — кнопки «Скопировать список» и «Скачать TXT»
  • Дублирующий красивый отчёт в консоли браузера
    (с console.table для удобной работы с данными).
  • Полностью локализованный интерфейс и логи на трёх языках.

│  ОТКУДА БРАТЬ CSV-ФАЙЛЫ                                         │
└─────────────────────────────────────────────────────────────────┘

  1. Перейдите на https://takeout.google.com под СТАРЫМ аккаунтом
     (тот, ОТКУДА переносите плейлисты).

  2. Нажмите «Снять выделение» — чтобы не качать всё подряд.

  3. Найдите в списке «YouTube и YouTube Music», поставьте галочку.

  4. Нажмите «Все данные YouTube» под этим пунктом →
     снимите всё → оставьте только «Плейлисты» → ОК.

  5. Прокрутите вниз → «Следующий шаг» → выберите формат и
     способ доставки → «Создать экспорт».

  6. Дождитесь письма (от пары минут до часа) → скачайте
     архив → распакуйте.

  7. Внутри найдите папку «YouTube и YouTube Music / Плейлисты»
     (или playlists/) — там будут файлы вида:
        «Видео в плейлисте _Название плейлиста_.csv»

Эти CSV-файлы и есть то, что нужно скармливать скрипту.


┌─────────────────────────────────────────────────────────────────┐
│  ФОРМАТ CSV И ИМЯ ПЛЕЙЛИСТА                                     │
└─────────────────────────────────────────────────────────────────┘

Скрипт ожидает CSV в формате Google Takeout:

    Идентификатор видео,Дата создания видео в плейлисте
    dQw4w9WgXcQ,2024-01-15T10:00:00+00:00
    9bZkp7q19f0,2024-01-15T10:05:00+00:00
    ...

Первая строка — заголовок, пропускается. В каждой следующей строке
берётся первая колонка (ID видео), всё после запятой игнорируется.
Пустые строки пропускаются.

Имя плейлиста определяется по имени файла:
  • Если в имени есть участок между подчёркиваниями вида _NAME_,
    в качестве названия плейлиста берётся NAME.
    Пример:  «Видео в плейлисте _Мой топ_.csv» → «Мой топ»
  • Иначе используется имя файла без расширения.

Если имя плейлиста (case-insensitive) равно «watch later»,
«смотреть позже», «посмотреть позже», «дивитися пізніше» или
«дивитись пізніше» — видео добавятся в системный «Смотреть позже»
целевого аккаунта, а не в новый одноимённый плейлист.


┌─────────────────────────────────────────────────────────────────┐
│  КАК ЗАПУСТИТЬ                                                  │
└─────────────────────────────────────────────────────────────────┘

  1. Войдите в новый YouTube-аккаунт (тот, КУДА переносите).
     Убедитесь, что в правом верхнем углу аватарка именно
     нужного аккаунта.

  2. Откройте https://www.youtube.com/ — любую страницу
     (главная подойдёт).

  3. Нажмите F12 → перейдите на вкладку «Console» (Консоль).

  4. Если консоль предупреждает «Don't paste anything here» /
     «Не вставляйте сюда ничего» — введите вручную:
            allow pasting
     и нажмите Enter. (Это защита YouTube/Chrome от self-XSS.)

  5. Откройте файл  import-playlists.js  в любом редакторе
     (Блокнот / VS Code / что угодно), выделите всё содержимое
     (Ctrl+A) и скопируйте (Ctrl+C).

  6. Вставьте в консоль (Ctrl+V) и нажмите Enter.

  7. В центре страницы появится модальное окно. При необходимости
     переключите язык кнопками EN / RU / UA в шапке.
     Перетащите туда CSV-файлы или нажмите на область загрузки
     и выберите их.

  8. Настройте задержку, приватность и при необходимости
     включите чекбокс «Очистить Watch Later перед импортом»
     (см. раздел «Watch Later — особенности» ниже).

  9. Нажмите «Запустить импорт» и дождитесь окончания.
     Окно покажет прогресс, лог и в конце — итоговый отчёт.


┌─────────────────────────────────────────────────────────────────┐
│  WATCH LATER — ОСОБЕННОСТИ                                      │
└─────────────────────────────────────────────────────────────────┘

«Смотреть позже» — системный плейлист с двумя особенностями,
о которых стоит знать.

ПОРЯДОК ВИДЕО
  В Google Takeout видео в Watch Later экспортируются в порядке
  отображения на сайте (новейшее сверху). YouTube же сортирует
  WL по дате добавления (самое свежее наверху). Поэтому скрипт
  автоматически добавляет видео из CSV в ОБРАТНОМ порядке: самое
  старое из CSV — первым, самое новое — последним. В результате
  отображение в новом Watch Later совпадает с оригиналом.

  Обычных плейлистов это не касается — там CSV-порядок сохраняется
  как есть.

ОЧИСТКА ПЕРЕД ИМПОРТОМ
  На странице настроек есть чекбокс «Очистить Watch Later перед
  импортом». Если включить — скрипт сначала удалит ВСЕ видео из
  текущего «Смотреть позже», и только потом добавит туда видео
  из CSV. Удобно, если предыдущий импорт прошёл частично и хочется
  начать с чистого листа.

  Удаление работает постранично (YouTube отдаёт по ~100 видео за
  раз): скрипт читает страницу, удаляет всё на ней, читает следующую,
  и так пока WL не станет пустым. В логе видно прогресс с номером
  страницы и общим счётчиком удалённых видео.

  Без галочки скрипт просто добавит CSV-видео поверх существующих
  (внутри WL YouTube сам не допустит дубликатов).

YouTube не публикует официальных лимитов на скорость, поэтому
рекомендации эмпирические:

  Размер плейлиста       Безопасная задержка
  ──────────────────     ───────────────────
  до 50 видео            500–700 мс
  50–200 видео           1–3 секунды
  200–500 видео          5–10 секунд
  500+ видео             10–15 секунд
                         или дробить на несколько сессий

Чем больше задержка — тем медленнее, но безопаснее.

При слишком высокой скорости возможны:
  • Временный rate-limit (HTTP 429). Скрипт ловит это автоматически
    и ставит паузу 30–60 секунд.
  • Капча на сайте. Пройдёте её во вкладке вручную — скрипт
    продолжит со следующего запроса.
  • Временное ограничение действий аккаунта на 24 часа.

В крайне редких случаях массовые автоматизированные запросы
могут привести к блокировке аккаунта.

║  импортировать всё за один заход.                            ║

┌─────────────────────────────────────────────────────────────────┐
│  ЧТО ДЕЛАТЬ С НЕ ДОБАВЛЕННЫМИ ВИДЕО                             │
└─────────────────────────────────────────────────────────────────┘

В отчёте видео делятся на две категории:

  ⊘ НЕДОСТУПНЫ
    Сервер ответил HTTP 400 / FAILED_PRECONDITION. Это значит,
    что видео нельзя добавить в плейлист. Причины:
        — удалено автором или YouTube;
        — переведено в приватное;
        — заблокировано в вашей стране;
        — поставлено возрастное ограничение
          (а на новом аккаунте не подтверждён возраст);
        — автор отключил добавление в плейлисты.
    С такими видео ничего не сделать — они утрачены.
    Открыв ссылку, вы увидите «Видео недоступно» или подобное.

  ✗ ОШИБКИ
    Всё остальное: сетевые сбои, неизвестные коды, rate-limit,
    с которым не справились ретраи. Имеет смысл попробовать ещё
    раз для этих конкретных видео.

В отчёте есть кнопки:
  📋 Скопировать список — кладёт в буфер обмена все провалившиеся
                          видео в формате «[Плейлист] URL Причина».
  💾 Скачать TXT        — сохраняет тот же список файлом
                          failed-videos.txt.

? Безопасно ли это для аккаунта?
  Скрипт использует тот же API, что и сама страница youtube.com,
  через ваши же cookie. С точки зрения серверов это выглядит
  как очень активное использование сайта. При разумной задержке
  (≥1 секунда) проблем обычно не возникает. Для перестраховки
  запускайте с задержкой 10+ секунд и не пытайтесь импортировать
  всё за один заход.

? Что делать с дубликатами в плейлистах?
  YouTube разрешает дубликаты — если видео уже есть в плейлисте,
  оно добавится ещё раз. Если не хотите — заранее уберите
  дубликаты из CSV.

? Можно ли импортировать в УЖЕ существующий плейлист?
  В этой версии — нет. Скрипт всегда создаёт новый плейлист с
  именем из файла, даже если плейлист с таким именем уже есть.
  Если в прошлом запуске уже создался плейлист с этим именем —
  удалите его вручную перед повторным импортом, иначе получите
  дубликат. Watch Later — исключение (это системный плейлист WL,
  в него всегда добавляется поверх существующего; либо включите
  «Очистить Watch Later перед импортом» для чистого старта).

? Если остановить импорт на полпути — продолжит при повторном
  запуске?
  Нет, скрипт не отслеживает состояние между запусками. Каждый
  запуск с нуля создаёт новые плейлисты. Перед повторным запуском:
    — для обычных плейлистов: удалите частично созданные руками
      (иначе получите дубликат);
    — для Watch Later: включите галочку «Очистить Watch Later
      перед импортом» — скрипт сам всё подчистит.

? Что делает чекбокс «Очистить Watch Later перед импортом»?
  Перед добавлением CSV-видео в Watch Later скрипт удалит ВСЕ
  видео, которые там сейчас. На обычные плейлисты не влияет.
  Используйте, если предыдущий частичный импорт оставил мусор
  в WL и хотите начать чисто.

? Почему видео в Watch Later добавляются в обратном порядке?
  Так и задумано. Google Takeout экспортирует WL в порядке
  отображения (новейшее сверху), а YouTube сортирует WL по дате
  добавления. Если бы скрипт добавлял в CSV-порядке, новейшее из
  оригинала оказалось бы внизу. Реверс гарантирует, что новое WL
  выглядит как оригинал. Обычных плейлистов это не касается.

? Где результаты после закрытия модалки?
  В переменной  window.__importResult  в консоли. Доступна до
  перезагрузки страницы. Можно скопировать в JSON через:
            copy(JSON.stringify(window.__importResult, null, 2))

? Можно ли менять язык во время импорта?
  Нет. Кнопки языка блокируются в момент клика «Запустить импорт»
  и остаются заблокированными до закрытия модалки или возврата
  в режим настройки — чтобы лог и итоговый отчёт оставались на
  одном языке. Чтобы снова сменить язык, нажмите «↺ Новый импорт»
  на странице итогов — вернётесь к экрану настроек.

? Как удалить случайно добавленный CSV-файл?
  В каждой строке файла справа есть кнопка ×. Нажмите её —
  файл уберётся из очереди, остальные останутся на месте.

? Можно ли запустить ещё один импорт, не закрывая окно?
  Да. После завершения главная кнопка превращается в
  «↺ Новый импорт». Клик по ней очищает список файлов,
  лог, прогресс и возвращает вас к экрану настройки.

? Скрипт не запускается, говорит «Не удалось прочитать
  INNERTUBE_API_KEY». Что делать?
  Откройте именно главную страницу https://www.youtube.com/
  (не Studio, не Music). На некоторых страницах ytcfg недоступен.

? Не появилось окно после вставки в консоль.
  Скорее всего вы не на youtube.com или есть ошибка в консоли —
  посмотрите красные сообщения. Также возможно, что вы вставили
  не весь файл — скопируйте ещё раз через Ctrl+A → Ctrl+C.

? Можно ли импортировать сразу с нескольких старых аккаунтов?
  Да, загрузите все CSV в одну сессию — главное, чтобы имена
  плейлистов не совпадали (иначе один перепишет другой в списке
  загруженных).

API-эндпоинты YouTube InnerTube, которые использует скрипт:

  POST /youtubei/v1/playlist/create
       — создание нового плейлиста (с одним стартовым видео)

  POST /youtubei/v1/browse/edit_playlist
       — добавление (ACTION_ADD_VIDEO) или удаление
         (ACTION_REMOVE_VIDEO_BY_VIDEO_ID) видео в плейлисте,
         включая системный WL

  POST /youtubei/v1/browse  (browseId: VLWL)
       — чтение содержимого Watch Later
         (используется только при включённой опции «очистить WL»)

Аутентификация:
  SAPISIDHASH = SHA-1(timestamp + ' ' + SAPISID + ' ' + origin)
  Заголовок Authorization: SAPISIDHASH <ts>_<hash>
  Тот же механизм, что использует сама страница YouTube.

Cookies используются только текущей сессии браузера (через
credentials: 'include'). Никакие данные никуда не отправляются,
кроме как на www.youtube.com.

Хранение настроек:
  localStorage['yt-importer-lang'] — выбранный язык интерфейса.
  Других данных скрипт не сохраняет.

Совместимость с Trusted Types CSP YouTube:
  Весь DOM строится через document.createElement + appendChild,
  без innerHTML / DOMParser / других «sink»-методов парсинга HTML.
